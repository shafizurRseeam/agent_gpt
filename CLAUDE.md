## Project Overview

**AgentGPT** is a privacy-preserving hybrid agent system targeting an NDSS paper.
The core idea: a local model (LC) handles reasoning and privacy filtering on-device;
a cloud model (CLM) handles expensive search and generation. The system prevents
unnecessary personal data from reaching the cloud.

Run everything from the project root (`agent_gpt/`) using `uv run python ...`.
See `FILES.md` for a full file reference with standalone run commands.

```

      │
      ▼

      │

      ▼

```

## PrivScope — Core Design

### Payload decomposition
```
p_t  =  U_loc  ∪  U_med  ∪  C_t
```
- **U_loc** — direct identifiers (name, phone, email, SSN, IDs). Withheld from cloud entirely. Stored in a private binding table. Replaced with `[TYPE]` placeholders.
- **U_med** — mediation candidates (dates, symptoms, address, insurance, prior providers). Evaluated by scope control.
- **C_t** — surrounding text / connectives. Passes through unchanged as skeleton.

### Stage 2: Scope Control — retention rules
```
κ = 0  (span text found in r_t):  keep if  Rel(u_j, g_t) >= ρ_low
κ = 1  (LC-injected from state):  keep if  TaskGain(u_j, g_t) >= γ
```
- **κ detection** is provenance-based: `span.text.lower() in r_t.lower()` → κ=0; else κ=1 (`"lc_injected"`). No memory-trace lookup, no sentence heuristic.
- **Rel** is embedding cosine similarity on type-aware span representations (see `_span_repr()` in `scope_control.py`). Computed only for κ=0 spans.
- **TaskGain** — LLM continuous decimal score (0.00–1.00) for κ=1 spans; rule table fallback. Uses g_t and the containing sentence from the working payload. Higher = span more strongly improves delegated cloud reasoning. Prior providers, venues, and historical outputs get low scores unless the task explicitly requires continuity/comparison.
- **ρ_high removed**: LC-injected spans bypass the Rel gate entirely — short structured values (address, date, insurance) score low on embeddings despite being task-critical.
- **No type-forcing**: types do not override the Rel/TaskGain signal.

### Task frame g_t
- **g_t is derived from r_t** (the original user request), not from p_t.
- Using p_t to build g_t would be circular (filtering using criteria derived from the data being filtered).
- g_t is generated by the local LLM as a constraint-schema summary of the delegated task.
- Fallback (no LLM): span-type-derived constraint labels + symptom noun phrases.

### Stage 3a: Sensitivity Classification

Classifies each retained span (from Stage 2) as **PTH** (passthrough) or **CSS** (context-sensitive):

- **PTH** — released verbatim; the exact value is required for cloud-side reasoning.
- **CSS** — forwarded to Stage 3b abstraction; the exact form is more specific than needed, or becomes linkable when co-released with other retained spans.

Classification is **joint**: all retained spans are presented together in a single local-LM call so the model can reason about co-release linkability (a span innocuous in isolation may become CSS when co-released with a specific date, location, and insurance plan).

**`_RISK_HINTS`** — per-type risk priors passed into the prompt as structured context. These are priors, not hard overrides; the LLM makes the final PTH/CSS decision from the task frame and co-release context.

**Rule fallback** (no LLM): `_PTH_TYPES = {date, time, money, party_size, preference}` → PTH; everything else → CSS.

**Debug display**: Stage 3a output shows the payload with PTH spans verbatim and CSS spans annotated as `[css:text]`, pending Stage 3b abstraction.

### Default thresholds
- `ρ_low = 0.10`, `γ = 0.50`

### Debug Display Conventions

Each standalone shows the payload at every stage boundary:

- **Stage 1 annotated view** (`tagged`): `[U_LOC:name]`, `[U_MED:address]`, … — bucket and type per span.
- **Stage 1 → Stage 2 input** (`working`): U_loc replaced with `[TYPE]`; U_med spans verbatim.
- **Stage 2 → Stage 3a input**: reconstructed payload with only retained spans; U_loc as `[TYPE]`.
- **Stage 3a → Stage 3b input**: retained payload with PTH spans verbatim and CSS spans as `[css:text]`.

## Critical Constraints

- **`privacy/privacyscope.py` must never be modified.** It is the original PrivacyScope baseline used in all evaluations. The new system is `privacy/privscope.py` (different file).
- **`privacy/privscope.py`** is the new PrivScope governor under development. Same public API as `privacyscope.py` for drop-in replacement.
- Scope control is **hybrid**: deterministic for current-workflow spans; LLM-scored TaskGain for carryover spans. Never fully deterministic, never pure LLM.
- The local LLM is accessed via `llm/local_llm.py` → Ollama at `http://localhost:11434`. Default model: `llama3.2`.

## Profile Field Sets (span_extractor.py)

Profile matching uses two disjoint field sets, both detected by exact match only:

**`_PROFILE_DIRECT_IDS`** → U_loc (always withheld):
name, phone, email, dob, ssn, passport_number, driver_license, insurance_id, patient_id, credit_card, membership_id, frequent_flyer_number, hotel_loyalty_id, vehicle_plate

**`_PROFILE_SOFT_FIELDS`** → U_med (evaluated by scope control):
address, insurance (→ insurance_name), doctor_gender_preference (→ preference), dietary_preference (→ preference), seat_preference (→ preference)
